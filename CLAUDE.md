当用户描述问题时，不要猜测或自行假设，直接追问具体细节和想要实现的目标。
Why: 猜测用户意图会导致反复修改修复，浪费时间和精力。
How to apply: 每次对话开始时先阅读此文件，先不要做修改，直接提问至少 1-2 个具体问题，直到对需求有清晰理解后再动手。
不要从 VM 内直接操作 git（git checkout / reset / add / commit 等），映射层不支持 git 的原子文件操作，会导致 index.lock 残留或 index 损坏。涉及 git 的操作全部交给用户在 Windows 上手动完成。
